## 2. 快速开始

### 2.1 配置环境变量

```powershell
# Windows
Copy-Item .env.example .env
Copy-Item frontend\.env.local.example frontend\.env.local
# 编辑 .env：无密钥时保持 LLM_MOCK=true
```

```bash
# Linux / macOS
cp .env.example .env && cp frontend/.env.local.example frontend/.env.local
```

### 2.2 后端

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1          # Windows
# source .venv/bin/activate           # macOS/Linux
pip install -r requirements.txt
cd ..
# 知识库入库（首次 / 新增文档后执行）
.\backend\.venv\Scripts\python scripts\ingest_kb.py
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

验证：`GET http://localhost:8000/api/health` 返回 `{"status":"ok"}`

### 2.3 前端

```bash
cd frontend
npm install
npm run dev       # 默认 http://localhost:3000
```

### 2.4 一键脚本（Windows）

```powershell
.\scripts\start.ps1              # 一键启动（后台无窗口，日志见 storage/logs）
.\scripts\start.ps1 -ShowWindows # 调试：显示后端/前端控制台窗口
.\scripts\start.ps1 -NoBrowser   # 启动但不打开浏览器
.\stop.bat                       # 停止服务
.\scripts\dev.ps1                # 调试：显示控制台，不打开浏览器
```

## 5. 后端架构

### 5.1 请求生命周期

```
客户端请求
  → FastAPI (main.py)
  → CORS Middleware
  → Router (api/routes/)
  → Service Layer (services/)
  → LangGraph build_graph().ainvoke(state)
  → 各 Agent Node
      ├── RAG retrieve(query)
      ├── LLM chat(prompt)
      └── DB save/load (repository.py)
  → 返回 JSON / SSE
```

### 5.2 服务层

`app/services/chat_service.py`：
- `run_chat(user_id, message)` → 一次性 JSON 响应
- `stream_chat(user_id, message)` → 异步生成器，产出 `{event, data}` 字典，由路由层包装成 SSE

`app/services/resource_service.py`：
- `generate_resources(user_id, types, topic)` → 调用 `generate_router_node`，返回资源列表

### 5.3 配置单例

```python
from app.core.config import get_settings

settings = get_settings()   # lru_cache，进程内单例
settings.llm_mock           # bool
settings.spark_api_key      # str
```

## 6. 多智能体系统

### 6.1 LangGraph 图结构

```
START → supervisor → [条件路由] → profile / generate / path / tutor / eval → END
```

所有节点签名均为 `async def xxx_node(state: AgentState) -> dict`，返回值合并回 `AgentState`。

```python
# graph.py 核心
@lru_cache
def build_graph():
    graph = StateGraph(AgentState)
    graph.add_node("supervisor", supervisor_node)
    graph.add_node("profile",    profile_node)
    graph.add_node("generate",   generate_router_node)
    graph.add_node("path",       path_node)
    graph.add_node("tutor",      tutor_node)
    graph.add_node("eval",       eval_node)
    graph.add_edge(START, "supervisor")
    graph.add_conditional_edges("supervisor", _route_after_supervisor, {...})
    # 各节点直接连 END
    return graph.compile()
```

### 6.2 AgentState 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | `str` | 学生 ID（默认 `"demo"`） |
| `messages` | `list[dict]` | 对话历史（`role/content`） |
| `intent` | `str` | supervisor 分类结果 |
| `profile` | `dict` | 学习画像 JSON |
| `resources` | `list[dict]` | 本次生成的资源列表 |
| `path` | `dict` | 学习路径 |
| `rag_context` | `str` | 检索到的知识片段 |
| `topic` | `str` | 资源生成主题 |
| `resource_types` | `list[str]` | 请求生成的资源类型 |
| `reply` | `str` | 最终回复文本 |

### 6.3 Supervisor 意图分类

`supervisor.py` 通过关键词匹配将消息路由至节点：

| 关键词示例 | 目标节点 |
|------------|----------|
| 评估、测验结果、掌握度 | `eval` |
| 为什么、解释、辅导、? | `tutor` |
| 路径、计划、规划 | `path` |
| 生成、文档、思维导图、练习 | `generate` |
| 其他（默认） | `profile` |

可在 `supervisor.py` 的常量列表中扩展关键词，或替换为 LLM 分类（见下文「扩展」）。

### 6.4 新增 Agent 节点

1. 在 `backend/app/agents/nodes/` 新建 `xxx_agent.py`：
   ```python
   from app.agents.state import AgentState

   async def xxx_node(state: AgentState) -> dict:
       # 从 state 读取所需字段
       # 调用 LLM / RAG
       # 返回更新字段
       return {"reply": "...", "resources": [...]}
   ```
2. 在 `graph.py` 注册节点与边：
   ```python
   graph.add_node("xxx", xxx_node)
   graph.add_edge("xxx", END)
   # 在 conditional_edges 映射中加入 "xxx": "xxx"
   ```
3. 在 `supervisor.py` 的关键词表添加触发词。
4. 在 `api/routes/` 暴露独立接口（可选）。

## 7. RAG 知识库

### 7.1 知识库目录规范

```
data/knowledge_base/
└── ml_intro/
    ├── README.md          # 课程简介（可选）
    └── chapters/
        ├── 01-导论.md
        ├── 02-线性回归.md
        └── ...            # 按章节编号命名
```

### 7.2 入库流程

```bash
# 使用 backend venv 内的 Python
.\backend\.venv\Scripts\python scripts\ingest_kb.py
```

流程：`load_markdown_files` → 按 `chunk_size=800, overlap=120` 分块 → 写入 ChromaDB collection `ml_intro`。

元数据含：`chapter`（文件名）、`title`（首个 `#` 标题）。

### 7.3 检索逻辑（`rag/retriever.py`）

1. **ChromaDB 路径**：`collection.query(query_texts=[query], n_results=k)` 向量检索
2. **回退路径**：Chroma 不可用或集合为空时，使用关键词交集打分，返回前 `k` 条

```python
from app.rag.retriever import retrieve

chunks = await retrieve("梯度下降原理", k=5)
# [{"id": "...", "text": "...", "metadata": {"chapter": "02-线性回归.md"}}]
```

### 7.4 扩展知识库

1. 在 `data/knowledge_base/` 下新建课程目录（如 `deep_learning/`）。
2. 在 `data/knowledge_base/manifest.json` 中注册内置库（`id`、`path`、`collection`）。
3. 重启后端，`ensure_builtin_libraries()` 会自动切块入库；或运行 `scripts/ingest_kb.py`（单库 legacy）。

> 完整资料库上传、多库 RAG、生成模式说明见 **[07-资源库与生成管线.md](./07-资源库与生成管线.md)**。

> 各 `page.tsx` 仅作薄包装器（约 10 行），真实逻辑在 `components/pages/` 对应的 Content 组件中。

### 9.2 性能优化架构

#### 代码分割（Code Splitting）

所有 Content 组件通过 `next/dynamic` 懒加载，独立打包为异步 chunk：

```typescript
const ChatContent = dynamic(
  () => import("@/components/pages/ChatContent"),
  { ssr: false }
);
```

`next.config.mjs` 自定义 `splitChunks.cacheGroups`，antd / echarts / markdown 各自独立 chunk。

#### Keep-alive 路由

`AppShell` 维护 `mountedState: Set<NavRoute>`，首次访问的页面永不卸载：

```typescript
// 切换仅修改 CSS，无 React unmount/remount 开销
<div style={{ display: isActive ? "block" : "none" }}>
  <Component />
</div>
```

#### 登录页预加载

`LoginContent.tsx` 挂载时立即触发所有 chunk 的 `import()`：

```typescript
const PAGE_CHUNKS = [
  () => import("@/components/pages/ChatContent"),
  () => import("@/components/pages/ProfileContent"),
  // ...
  () => import("echarts"),
];
// 并行加载，进度条实时反映 loadedCount / totalChunks
```

#### 登录后初始化进度

`AppShell` 登录后跟踪 3 个 API + 5 个 chunk + ECharts 共 9 项任务，全部完成后才淡出遮罩。


## 10. API 契约

### `POST /api/chat`（JSON 响应）

请求：
```json
{
  "user_id": "demo",
  "message": "我想复习线性回归，基础一般"
}
```

响应：
```json
{
  "reply": "已更新学习画像...",
  "profile": { "knowledge_level": "入门", ... },
  "resources": [],
  "path": null
}
```

### `POST /api/chat/stream`（SSE）

SSE 事件流：

| 事件类型 | `data` 内容 |
|----------|-------------|
| `token` | 增量文本字符串 |
| `profile` | 更新后的画像 JSON |
| `resources` | 新生成资源列表 JSON |
| `done` | 空字符串 |
| `error` | 错误信息字符串 |

### `POST /api/resources/generate`

```json
{
  "user_id": "demo",
  "topic": "线性回归",
  "library_id": "builtin-ml-intro",
  "new_library_name": null,
  "resource_types": ["doc", "mindmap", "quiz", "reading", "media", "code"]
}
```

- 有 `library_id` 且资料库已入库：RAG + 可选少量全网补充
- 无资料库或空库：两步全网检索整理（LLM）后生成
- `new_library_name`：创建空资料库并关联本次生成（空库仍走全网模式）

流式接口 `POST /api/resources/generate/stream` 额外 SSE 阶段：`context`、`web_research`。

### 资料库 API（`/api/libraries`）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/libraries?user_id=` | 内置 + 用户资料库列表 |
| POST | `/api/libraries` | 创建空资料库 |
| POST | `/api/libraries/{id}/upload` | multipart 多文件上传分析 |
| GET | `/api/libraries/{id}` | 详情 |
| DELETE | `/api/libraries/{id}` | 删除用户库 |

详见 [07-资源库与生成管线.md](./07-资源库与生成管线.md)。

资源 `type` 枚举：`doc` / `mindmap` / `quiz` / `reading` / `media` / `code` / `ppt` / `design` / `project`

### `GET /api/profile/{user_id}`

返回 `StudentProfile` JSON，不存在时返回 `null`。

### `GET /api/path/{user_id}`

返回 `LearningPath` JSON，含 `steps` 列表（`title / description / resource_ids / duration / status`）。

### `POST /api/path/{user_id}/refresh`

重新规划路径，返回新 `LearningPath`。

## 12. 防幻觉与安全

`backend/app/core/guardrails.py`：

```python
filter_sensitive(text)              # 敏感词替换
attach_sources(content, chunks)     # 拼接引用脚注
review_consistency(generated, ctx)  # 关键词重叠检查
```

所有 Agent 在返回 `reply` 前调用 `filter_sensitive`；生成内容通过 `attach_sources` 附加来源。

扩展方向：在 `graph.py` 中于 DocAgent 后增加 `reviewer_node`，调用 LLM 对生成内容做一致性审核。

## 5. 多智能体开发流程

### 5.1 新增 Agent 节点

1. 在 `backend/app/agents/nodes/` 新建 `xxx_agent.py`，实现 `async def run(state) -> dict`。  
2. 在 `backend/app/agents/graph.py` 注册节点与边。  
3. 在 `supervisor.py` 的意图路由表增加映射。  
4. 在 `api/routes/` 暴露触发接口（若需独立 API）。  
5. 更新 `docs/01-需求规格说明书.md` 资源类型表。

### 5.2 状态结构

主编排使用 `AgentState`（`backend/app/agents/state.py`）：

- `messages`：对话历史  
- `user_id`：学生 ID  
- `intent`：supervisor 分类结果  
- `profile`：画像 JSON  
- `resources`：本次生成的资源列表  
- `path`：学习路径步骤  
- `rag_context`：检索片段  

### 5.3 LangGraph 执行

```python
from app.agents.graph import build_graph

graph = build_graph()
result = await graph.ainvoke({"user_id": "demo", "messages": [...]})
```

服务层封装：`app/services/chat_service.py`、`resource_service.py`。

## 6. RAG 知识库流程

1. 将章节 Markdown 放入 `data/knowledge_base/ml_intro/chapters/`。  
2. 运行 `python scripts/ingest_kb.py`：分块 → embedding（框架阶段可用简单 hash embedding 或 sentence-transformers）→ 写入 Chroma。  
3. Agent 通过 `app/rag/retriever.py` 的 `retrieve(query, k=5)` 获取上下文。  
4. 生成结果必须携带 `sources` 字段（chunk id / 章节名）。

**分块建议**：chunk_size=800，overlap=120；元数据含 `chapter`、`title`。
