# 07 — 资源库与生成管线

**版本**：v1.0 | **更新**：2026-05-23

本文说明「课程资料库」与「多模态学习资源生成」的端到端设计，对齐赛题要求：**自建课程知识库输入**、**大模型依据资料生成**、**无资料时全网检索整理**。

---

## 1. 概念区分

| 概念 | 说明 | 存储 |
|------|------|------|
| **课程资料库** | 原始课件/讲义/代码等，经解析 + LLM 分析 + 向量入库 | SQLite `resource_libraries` + Chroma 集合 |
| **学习资源** | 系统生成的 doc/quiz/mindmap 等多类型成品 | SQLite `learning_resources` |
| **内置资料库** | 仓库 `data/knowledge_base/` 预置课程 | manifest.json 注册，启动时自动入库 |
| **用户资料库** | 用户上传文件创建 | `storage/` 元数据 + 独立 Chroma 集合 |

---

## 2. 内置资料库

路径：`data/knowledge_base/manifest.json`

当前预置：

1. **机器学习导论**（`builtin-ml-intro`）— 5 章 Markdown
2. **Python 程序设计基础**（`builtin-python-basics`）— 2 章 Markdown

启动后端时 `ensure_builtin_libraries()` 会：

- 读取 manifest
- 将章节切块写入 Chroma（集合名见 manifest `collection` 字段）
- 在数据库注册 `source_type=builtin`

## 3. 用户上传资料库

### 3.1 支持格式

`.md` `.txt` `.pdf` `.docx` 及常见代码/数据扩展名（见 `GET /api/libraries/supported-formats`）。

### 3.2 处理流水线（多步 LLM）

```mermaid
flowchart LR
  A[多文件上传] --> B[文本提取]
  B --> C[LLM 单文件分析]
  C --> D[LLM 资料库合成]
  D --> E[分块 + Chroma 入库]
  E --> F[资料库 status=ready]
```

| 步骤 | Prompt | 模型通道 |
|------|--------|----------|
| 单文件分析 | `LIBRARY_FILE_ANALYSIS_SYSTEM` | Primary（星火） |
| 库级合成 | `LIBRARY_SYNTHESIS_SYSTEM` | Primary |
| 分块入库 | — | Chroma |

Prompt 定义见 `backend/app/core/prompts.py`。

### 3.3 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/libraries` | 列出内置 + 当前用户资料库 |
| POST | `/api/libraries` | 创建空资料库 |
| POST | `/api/libraries/{id}/upload` | 多文件上传并分析入库 |
| GET | `/api/libraries/{id}` | 详情 + 文件列表 |
| DELETE | `/api/libraries/{id}` | 删除用户资料库 |

## 4. 资源生成模式

生成请求 `POST /api/resources/generate/stream` 扩展字段：

```json
{
  "user_id": "demo",
  "topic": "线性回归",
  "library_id": "builtin-ml-intro",
  "new_library_name": null,
  "resource_types": ["doc", "mindmap", "quiz"]
}
```

| 场景 | 条件 | 上下文策略 | SSE 阶段 |
|------|------|------------|----------|
| **资料库模式** | 选中资料库且 `chunk_count > 0` | RAG 检索 k=6；片段不足时 1 次 LLM 补充 | `context` → 各类型 → `reviewer` |
| **资料库+补充** | 有库但片段偏少 | 库内 RAG + `WEB_SUPPLEMENT` | `context` → … |
| **全网模式** | 未选库 / 空库 / 显式「无资料库」 | `plan_web_research` → `synthesize_web_research`（2 次 Primary LLM） | `context` → `web_research` → … |

各资源类型由 `_build_resource` 调用 Primary LLM，System Prompt 为 `resource_generation_system(type)`，User Prompt 注入资料库片段、全网摘要、学生画像摘要。

无 LLM Key 时（`LLM_MOCK=true`）回退模板 + Mock 占位正文。

## 5. 前端交互

资源库页两个 Tab：

1. **学习资源** — 按学习阶段 + 类型浏览已生成资源（原有旅程视图）
2. **课程资料库** — 卡片管理、上传、选中

页头生成区：

- **生成依据**：依据资料库 / 新建资料库 / 无资料库·全网检索
- **主题** + **生成新资源 / 完整套件**

---

## 6. 防幻觉与安全

- 资料库分析 Prompt 要求「仅基于文件内容」
- 全网整理 Prompt 要求标注可信度、禁止伪造 DOI/URL
- 生成后仍走 `filter_sensitive` + `review_consistency` + `reviewer_agent`（Aux LLM）

## 8. 测试建议

1. 打开「课程资料库」Tab，确认两个内置库 `chunk_count > 0`
2. 新建资料库 → 上传 `.md`/`.pdf` → 观察分析完成与片段数增加
3. 选择「机器学习导论」生成 → SSE 出现 `context`，无 `web_research`
4. 选择「无资料库·全网检索」→ SSE 出现 `web_research`
5. 检查生成资源文末「生成依据」脚注
