# 学径（LearnPath）需求规格说明书

**版本**：v0.1  
**日期**：2026-05-22  
**赛题**：A3 — 基于大模型的个性化资源生成与学习多智能体系统开发  
**默认课程**：机器学习 / 人工智能导论（`ml_intro`）

---

## 1. 文档目的与范围

本文档定义「学径」系统的业务目标、用户角色、功能与非功能需求及验收标准，作为设计、开发、测试与初赛答辩的统一依据。

**范围**：Web 端多智能体学习系统（对话画像、资源生成、学习路径、可选辅导与评估）。  
**范围外（本期框架阶段）**：真实视频合成、移动端、生产级集群部署。

---

## 2. 背景与问题陈述

高等教育学生面临资源繁杂、难以匹配个人进度、缺乏个性化指导等问题。学径以**一门完整专业课程**为切入点，通过**多智能体协同**与**课程知识库 RAG**，为学生生成可 grounding 的多模态学习资源，并动态规划学习路径。

### 2.1 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 学生 | 本科/研究生，学习 ML 导论等课程 | 少填表、多对话；资源贴合短板；路径清晰 |
| 指导教师（可选） | 课程负责人 | 审核知识库与生成内容质量 |
| 评审/演示者 | 大赛评委 | 可复现运行、流程清晰、文档完整 |

### 2.2 项目目标

1. 提升学习效率：精准资源与路径，减少无效检索时间。  
2. 优化资源利用：按画像推送已生成资源。  
3. 增强学习效果：练习、评估与辅导闭环（加分项）。

## 4. 功能需求

### 4.1 FR-01 对话式学习画像自主构建（P0）

**描述**：通过多轮对话抽取并维护学生画像，维度不少于 6 个，支持随学随新。

| 维度 ID | 字段名 | 说明 | 更新触发 |
|---------|--------|------|----------|
| D1 | knowledge_level | 知识基础（入门/进阶/有项目经验） | 对话、测验结果 |
| D2 | learning_goal | 学习目标（考试/项目/科研） | 对话 |
| D3 | cognitive_style | 认知风格（偏理论/偏实践/视觉型等） | 对话、资源偏好 |
| D4 | error_prone_topics | 易错/薄弱知识点列表 | 练习、对话 |
| D5 | preferred_modality | 偏好模态（文档/视频/练习/代码） | 行为、对话 |
| D6 | pace_and_time | 学习节奏与每周可投入时间 | 对话 |
| D7 | recent_progress | 近期进度与掌握度摘要（扩展维） | 路径、测验 |

**输入**：用户自然语言消息、可选历史会话。  
**输出**：结构化 `StudentProfile` JSON，持久化至数据库。  
**处理**：`ProfileAgent` 调用星火 LLM + 规则校验，合并增量更新。

**验收标准**：
- [ ] 新用户 3–5 轮对话后可展示 ≥6 维画像字段  
- [ ] 后续对话可更新至少 1 个维度且保留历史版本或更新时间戳  
- [ ] 前端画像面板可读展示各维度

---

### 4.2 FR-02 多智能体协同的资源生成（P0）

**描述**：不同角色 Agent 协作生成至少 5 类个性化资源，内容须基于课程知识库 RAG。

| 资源类型 | 负责 Agent | 输出格式 | 最低验收 |
|----------|------------|----------|----------|
| 专业课程讲解文档 | DocAgent | Markdown | 含章节结构、引用来源 |
| 知识点思维导图 | MindmapAgent | Mermaid / JSON | 覆盖指定知识点 |
| 练习题 | QuizAgent | JSON（题干+选项+解析） | ≥3 题/次 |
| 拓展阅读 | ReadingAgent | Markdown + 链接摘要 | 与知识点相关 |
| 多模态教学说明 | MediaAgent | 图文卡片 + 分镜脚本 | 可演示「类视频」脚本 |
| 代码实操案例 | CodeAgent | Markdown + 代码块 | 可运行或伪代码+说明 |

**编排**：`Supervisor` 根据用户意图路由至对应节点；可串行「生成 → Reviewer 质检」。  
**验收标准**：
- [ ] 单次请求可触发 ≥1 类资源生成并返回结构化结果  
- [ ] 演示中累计展示 ≥5 类资源  
- [ ] 生成内容含 `source_chunks` 或引用标识（防幻觉）

---

### 4.3 FR-03 个性化学习路径规划与资源推送（P0）

**描述**：`PathAgent` 综合画像、掌握度、已生成资源，输出有序学习步骤并推送。

**输入**：`StudentProfile`、资源元数据列表、可选测验结果。  
**输出**：`LearningPath`（步骤列表：标题、目标、关联资源 ID、建议时长、状态）。  

**验收标准**：
- [ ] 路径至少 3 步且顺序合理（依赖前置知识）  
- [ ] 每步可关联已生成资源  
- [ ] 前端路径页展示时间线/步骤卡

---

### 4.4 FR-04 智能辅导（P1，加分）

**描述**：`TutorAgent` 对学习问题提供文字解答、Mermaid 图解、短视频分镜脚本。

**验收标准**：
- [ ] 问答流式返回  
- [ ] 回答引用知识库片段或明确标注不确定

---

### 4.5 FR-05 学习效果评估（P1，加分）

**描述**：`EvalAgent` 跟踪练习正确率、资源完成度，输出评估报告并触发路径/推送调整。

**验收标准**：
- [ ] 提交测验后生成简要评估  
- [ ] 评估结果可驱动 `PathAgent` 更新推荐（接口预留）

---

## 5. 非功能需求

| ID | 类别 | 要求 |
|----|------|------|
| NFR-01 | 易用性 | 界面简洁；对话流式输出；Markdown/卡片展示资源 |
| NFR-02 | 性能 | 对话首 token 流式可见；长任务展示生成进度或阶段事件 |
| NFR-03 | 安全 | 敏感词过滤；学术内容 RAG grounding；低置信度不编造 |
| NFR-04 | 合规 | 开源组件在 `docs/03-开源参考与协议.md` 标注；AI 工具使用讯飞相关能力 |
| NFR-05 | 可维护 | 模块化 Agent；环境变量配置 LLM；Mock 模式可无密钥启动 |
| NFR-06 | 可部署 | 提供 README 启动说明；常规 Win/Linux + Python 3.11+ / Node 20+ |

## 6. 数据需求

### 6.1 知识库

- 路径：`data/knowledge_base/ml_intro/`  
- 格式：Markdown 章节文件，含 front matter（chapter, title, tags）  
- 规模（目标）：≥5 章完整讲义；框架阶段提供 2 章样例  

### 6.2 持久化实体

| 实体 | 主要字段 |
|------|----------|
| StudentProfile | user_id, dimensions(JSON), updated_at |
| LearningResource | id, type, title, content_path, sources, created_at |
| LearningPath | user_id, steps(JSON), version |
| ChatMessage | session_id, role, content, timestamp |

## 7. 外部接口需求（API 概要）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| POST | `/api/chat` | 对话（支持 SSE 流） |
| GET | `/api/profile/{user_id}` | 获取画像 |
| POST | `/api/resources/generate` | 触发资源生成 |
| GET | `/api/resources` | 资源列表 |
| GET | `/api/path/{user_id}` | 学习路径 |
| POST | `/api/tutor/ask` | 辅导问答（P1） |
| POST | `/api/eval/submit` | 提交评估（P1） |

详细契约见 `docs/02-开发指南.md` 与后端 OpenAPI（`/docs`）。

## 8. 约束与假设

1. **LLM**：默认讯飞星火 HTTP 兼容接口；无密钥时使用 Mock。  
2. **多智能体框架**：LangGraph 状态图，须在文档与答辩中说明协同关系。  
3. **多模态视频**：初赛以分镜脚本 + 图文卡片代替完整视频合成。  
4. **测试数据**：团队自建 `ml_intro` 知识库，不依赖赛方数据集。
