> 版本：v1.0  
> 出题企业：科大讯飞股份有限公司  
> 文档状态：草稿

---

## 1. 项目概述

### 1.1 背景

在高等教育领域，学生面临"学习资源繁杂、无法个性化适配、缺乏智能化引导"三大核心痛点。LearnPath 系统借助大模型技术（讯飞星火 Spark API）与多智能体协同框架（LangGraph），以人工智能专业课程为切入点，构建可自动生成个性化、多模态学习资源的多智能体系统，实现"因材施教"的数字化落地。

### 1.2 目标用户

- **主要用户**：高等院校本科生、研究生（AI/CS/电子信息相关专业）
- **次要用户**：课程教师（查看学生画像与学习进度）

### 1.3 系统定位

面向高等教育场景的 **AI 学习助理平台**，核心价值：
1. 通过对话自动构建动态学生画像
2. 多智能体协同生成≥5类多模态个性化学习资源
3. 规划科学学习路径并精准推送资源

## 2. 功能需求

### 2.1 F1 — 对话式学习画像构建（必须实现）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F1-1 | 系统通过多轮自然语言对话（≥5轮）收集学生信息 | P0 |
| F1-2 | 自动抽取并构建包含≥6个维度的动态学生画像 | P0 |
| F1-3 | 画像维度包括：知识基础、认知风格、学习目标、易错点、可用时间、兴趣方向 | P0 |
| F1-4 | 支持画像随学习进展动态更新（随学随新） | P1 |
| F1-5 | 可视化展示学生画像（雷达图/维度卡片） | P1 |

**画像数据结构（JSON Schema）：**
```json
{
  "student_id": "string",
  "knowledge_level": { "score": 0-10, "weak_points": ["..."] },
  "cognitive_style": "visual|auditory|reading|kinesthetic",
  "learning_goal": "string",
  "error_prone_topics": ["..."],
  "available_time_per_day": "hours",
  "interest_direction": ["..."],
  "updated_at": "timestamp"
}
```

---

### 2.2 F2 — 多智能体协同资源生成（必须实现）

系统须由**不同角色智能体**协作，生成≥5种类型的个性化资源：

| 编号 | 资源类型 | 负责智能体 | 优先级 |
|------|----------|------------|--------|
| F2-1 | 专业课程讲解文档（Markdown/PDF） | DocumentAgent | P0 |
| F2-2 | 知识点思维导图（JSON/SVG） | MindMapAgent | P0 |
| F2-3 | 个性化练习题库（选择/填空/编程） | QuizAgent | P0 |
| F2-4 | 多模态教学视频脚本 + 动画提示词 | VideoAgent | P0 |
| F2-5 | 代码类实操案例（含注释+讲解） | CodeAgent | P0 |
| F2-6 | 拓展阅读材料（摘要+链接） | ResourceAgent | P1 |

**智能体协作流程：**
```
用户请求 → OrchestratorAgent
             ├── ProfileAgent（更新画像）
             ├── CurriculumAgent（规划学习路径）
             └── ResourceAgents（并行生成资源）
                  ├── DocumentAgent
                  ├── MindMapAgent
                  ├── QuizAgent
                  ├── VideoAgent
                  └── CodeAgent
```

## 4. 系统用例

### UC-01：初次使用 - 构建学习画像

**主角**：新学生  
**前置条件**：已注册登录  
**主流程**：
1. 学生进入对话界面
2. ProfileAgent 发起引导性问题（专业、学习目标等）
3. 学生通过自然语言回答
4. 系统自动提取信息并更新画像（≥6维度）
5. 系统展示已构建的画像卡片并询问是否正确

---

### UC-02：生成个性化学习资源包

**主角**：已有画像的学生  
**前置条件**：学生画像存在  
**主流程**：
1. 学生输入"我要学习神经网络反向传播"
2. OrchestratorAgent 解析意图并拆解任务
3. 多个 ResourceAgent 并行工作：
   - 生成讲解文档
   - 生成思维导图数据
   - 生成3道练习题
   - 生成代码实例
4. 前端分批展示生成结果（流式卡片）
5. 学生可下载或收藏资源

---

### UC-03：规划学习路径

**主角**：学生  
**主流程**：
1. 学生请求"帮我制定一个月的神经网络学习计划"
2. CurriculumAgent 结合画像与知识库生成路径
3. 前端展示可交互的学习路径时间线
4. 学生标记完成，系统动态调整后续计划

---

## 5. 数据需求

- **学生画像数据**：存储于 PostgreSQL/SQLite
- **学习资源数据**：生成后存储为文件 + 数据库索引
- **知识库文档**：向量化存储于 ChromaDB（RAG 支持）
- **学习行为日志**：时序数据，存于数据库

---

## 6. 约束与假设

- **LLM 约束**：必须使用讯飞星火 API（`SparkAPI`）；推荐使用 `4.0 Ultra` 版本
- **多智能体框架**：LangGraph（MIT License）
- **知识库**：自行构造至少一门完整高校专业课程（本项目选用：人工智能基础课程）
- **部署**：支持 Docker Compose 一键部署

## 7. 验收标准

| 功能 | 验收标准 |
|------|----------|
| 学生画像 | 通过≥5轮对话自动生成包含6+维度的画像 |
| 资源生成 | 一次请求可生成≥5种类型的资源，各类型内容完整 |
| 学习路径 | 生成的路径包含≥3阶段，每阶段有明确目标和资源 |
| 界面 | 支持流式输出、Markdown渲染、无明显功能错误 |
| 安全 | 生成内容无明显事实性错误（知识库RAG溯源验证） |
